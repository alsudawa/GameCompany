# Project Brief — void-pulse

**Status:** draft
**Producer:** @producer
**Date:** 2026-04-17

## Target player

Casual player with 30–90 seconds to burn — commuter, break-room visitor, someone idly unlocking their phone. Expects immediate gratification, zero reading required.

## Session goal

Target session: 45–75 seconds. Target retries per session: 3–6. The game should feel "just one more go" from the first death.

## Platform

Web (desktop + mobile), single input (pointer/touch).

## Success criteria

- [ ] Player understands the game in under 60s (no tutorial screen)
- [ ] Game-over to next run in under 3s
- [ ] First playtest wants to retry within 60s

## Scope statement

A single-screen survival game built around escalating visual rhythm and one-tap interaction. The entire threat model, feedback, and scoring must be legible at a glance — no text instructions, no menus beyond a start tap and an instant-restart tap. Out of scope: multiple levels, unlocks, persistent save data, external assets of any kind.
